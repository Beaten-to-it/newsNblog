- **어디에 쓰나:** 제약·바이오 R&D / 재료과학 / 실험 자동화 팀
- **어떻게 적용하나:** 모델에게 문헌 조사와 후보 가설 생성을 맡기되, 실험 후보 선택·실험실 실행·최종 해석은 도메인 전문가가 승인하는 구조로 설계합니다. 실험 자동화 장비가 있다면 후보 space를 넓게 탐색하고, 대표 샘플만 인간 연구자가 재현해 검증하는 방식이 현실적입니다.
- **주의할 점:** 하나의 반응군에서 나온 개선을 일반화하면 안 됩니다. 위험한 생물·화학 응용은 모델 차단, 접근 통제, 실험실 승인, 결과 재현 절차가 함께 있어야 합니다.

### 2. [사내 에이전트 도구를 ARD/registry 방식으로 정리하기](https://github.blog/changelog/2026-06-17-agent-finder-for-github-copilot-now-available/)

- **어디에 쓰나:** 개발 플랫폼 / 엔터프라이즈 AI / 보안 거버넌스
- **어떻게 적용하나:** 사내 MCP 서버, 코드 검색, 배포 스크립트, 데이터 조회 도구를 하나의 registry에 등록하고, 각 도구에 설명·입력·권한·운영자·사용 조건을 붙입니다. 에이전트가 모든 도구를 미리 들고 시작하지 않고, 작업 설명에 맞춰 필요한 도구만 찾도록 만들면 비용과 보안 노출을 줄일 수 있습니다.
- **주의할 점:** discovery는 authorization을 대체하지 않습니다. 도구가 검색된 뒤에도 사용자 권한, 환경, 변경 범위, 감사 로그, 승인 단계가 별도로 적용돼야 합니다.
