## 3. 오늘 꼭 봐야 할 AI 뉴스 Top 5

### 1. [OpenAI: Introducing Lockdown Mode and Elevated Risk labels in ChatGPT](https://openai.com/index/introducing-lockdown-mode-and-elevated-risk-labels-in-chatgpt/)

**핵심 요약**  
OpenAI는 6월 4일 업데이트를 통해 Lockdown Mode를 개인 ChatGPT 계정과 self-serve ChatGPT Business 계정으로 확대 배포한다고 밝혔습니다. 이 모드는 live web access, Deep Research, Agent Mode, live connectors, 파일 다운로드 등 외부 서비스와 연결되는 기능을 제한하거나 끕니다.

**왜 중요한가**  
Prompt injection 방어는 “모델이 악성 지시를 무시하게 만들기”만으로 끝나지 않습니다. 민감한 데이터가 있는 조직은 AI가 외부 콘텐츠를 읽고, 내부 데이터를 참고하고, 네트워크로 내보낼 수 있는 도구 조합을 끊는 운영 모드가 필요합니다.

**태그**  
`OpenAI` `ChatGPT` `prompt injection` `AI security`

### 2. [TechCrunch: OpenAI unveils Lockdown Mode to protect sensitive data from prompt injection attacks](https://techcrunch.com/2026/06/06/openai-unveils-lockdown-mode-to-protect-sensitive-data-from-prompt-injection-attacks/)

**핵심 요약**  
TechCrunch는 Lockdown Mode가 민감한 데이터를 다루는 사용자·조직을 위한 기능이며, live browsing 대신 cached content로 제한하고 외부 이미지 검색, Deep Research, Agent Mode를 제한한다고 정리했습니다. 동시에 OpenAI도 cached web content나 업로드 파일 안의 prompt injection이 여전히 응답 품질과 행동에 영향을 줄 수 있다고 경고합니다.

**왜 중요한가**  
이 보도는 보안 기능의 한계까지 같이 보여줍니다. 기업이 ChatGPT나 AI 에이전트를 도입할 때 “보안 모드 켰다”에서 끝내지 말고, 업로드 파일 검증, 데이터 분류, 커넥터 권한, 로그 감사까지 같이 설계해야 합니다.

**태그**  
`OpenAI` `TechCrunch` `enterprise security` `data exfiltration`

### 3. [Google: Introducing Gemma 4 12B](https://blog.google/innovation-and-ai/technology/developers-tools/introducing-gemma-4-12b/)

**핵심 요약**  
Google은 Gemma 4 12B를 “노트북에서 agentic multimodal intelligence를 실행”하기 위한 12B급 모델로 소개했습니다. 16GB RAM/VRAM/unified memory 환경을 겨냥하고, vision/audio encoder를 줄인 통합 구조와 native audio 입력, MTP drafter, Apache 2.0 라이선스를 강조합니다.

**왜 중요한가**  
클라우드 frontier 모델이 모든 문제의 답은 아닙니다. 개인정보, 오프라인, 비용, 지연시간이 중요한 업무에서는 12B급 로컬 멀티모달 모델이 실제 제품 설계의 강력한 선택지가 됩니다.

**태그**  
`Google` `Gemma` `local AI` `multimodal`

### 4. [Ladybird: Changing How We Develop Ladybird](https://ladybird.org/posts/changing-how-we-develop-ladybird/)

**핵심 요약**  
Ladybird는 앞으로 공개 pull request를 받지 않고, 코드 변경은 프로젝트 maintainer만 도입한다고 발표했습니다. 이유는 첫 alpha와 실제 사용자용 브라우저 출시를 준비하며 보안 모델과 책임 경계를 강화해야 하고, AI 도구가 큰 패치를 빠르고 싸게 만들면서 PR 자체가 더 이상 기여자의 노력과 신뢰를 충분히 보여주지 못한다는 판단입니다.

**왜 중요한가**  
AI 코딩 시대의 오픈소스는 기여량 증가가 곧 건강함을 뜻하지 않습니다. 특히 브라우저·백업·보안 도구처럼 공격면이 큰 프로젝트는 “누가 코드를 썼나”보다 “누가 책임지고 유지할 수 있나”를 기준으로 프로세스를 바꾸게 됩니다.

**태그**  
`Open Source` `AI coding` `security` `governance`
