### 3. [Introducing computer use in Gemini 3.5 Flash](https://blog.google/innovation-and-ai/models-and-research/gemini-models/introducing-computer-use-gemini-3-5-flash/)

## 4. AI UseCase

> 오늘 뉴스에서 바로 참고할 만한 실제 적용 아이디어만 짚습니다.

### 1. [AI 서비스의 추론 비용 지표를 제품 KPI로 올리기](https://openai.com/index/openai-broadcom-jalapeno-inference-chip/)

- **어디에 쓰나:** AI SaaS / 내부 Copilot / 대규모 챗봇 운영 / API 사업
- **어떻게 적용하나:** 모델 품질 지표와 함께 토큰당 원가, 요청당 지연 시간, peak hour capacity, fallback 모델 비용, cache hit ratio를 제품 대시보드에 넣습니다. 고비용 기능은 "사용자 가치가 큰 단계"와 "저렴한 모델로 충분한 단계"를 나눠 라우팅합니다.
- **주의할 점:** 자체 칩 뉴스가 곧 모든 고객 비용 절감으로 이어지는 것은 아닙니다. 기술 보고서, 실제 배포 시점, provider별 가격 정책, workload별 효율 차이를 분리해서 봐야 합니다.

### 2. [Computer-use 에이전트로 웹 앱 회귀 QA와 접근성 점검 자동화하기](https://blog.google/innovation-and-ai/models-and-research/gemini-models/introducing-computer-use-gemini-3-5-flash/)

- **어디에 쓰나:** QA / accessibility audit / SaaS 운영 / 내부 업무 자동화
- **어떻게 적용하나:** 에이전트가 실제 브라우저에서 로그인, 화면 이동, 폼 입력, 오류 메시지 확인, keyboard navigation 점검을 수행하게 합니다. 성공 여부는 스크린샷, DOM 상태, 로그, 재현 단계로 남기고, 결제·삭제·권한 변경처럼 되돌리기 어려운 작업은 반드시 사람 확인을 요구합니다.
- **주의할 점:** 페이지 안의 텍스트가 에이전트에게 명령처럼 작동할 수 있습니다. sandbox, allowlist, 민감 액션 confirmation, prompt injection 감지, 권한이 낮은 test account를 함께 써야 합니다.

### 3. [팀 채널형 에이전트를 업무 위임보다 "권한 설계 실험"으로 먼저 도입하기](https://www.anthropic.com/news/introducing-claude-tag)

- **어디에 쓰나:** Slack 기반 개발팀 / 지원팀 / PM 조직 / 데이터 분석 요청
- **어떻게 적용하나:** 처음부터 모든 채널에 붙이지 말고, private test channel에서 티켓 초안 작성, 회의 후 action item 정리, 간단한 로그 조회처럼 좁은 업무만 맡깁니다. 채널별 Claude identity, 연결 도구, 월간 비용 한도, action log 리뷰 담당자를 미리 정합니다.
- **주의할 점:** 공유 채널의 agent는 개인 권한을 그대로 대체하지 않습니다. 누가 요청했는지, agent가 어떤 credential로 행동하는지, 다른 사람이 thread에 끼어들 때 작업이 어떻게 바뀌는지 명확해야 합니다.
