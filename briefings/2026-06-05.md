- **어떻게 적용하나:** 작은 단위의 작업 카드로 쪼개고, Claude Code가 수정안을 만들면 사람이 PR 리뷰/테스트 결과를 확인하는 방식으로 운영합니다.
- **주의할 점:** 사용량 한도와 비용을 먼저 계산하고, secrets나 배포 권한이 있는 환경에서는 에이전트 권한을 제한해야 합니다.

### 2. [검색/리서치를 에이전트형 모니터링 루프로 전환](https://blog.google/products-and-platforms/products/search/search-io-2026/)

- **어디에 쓰나:** 경쟁사 모니터링, 가격 추적, 채용/시장 조사, 제품 리서치
- **어떻게 적용하나:** 사람이 매번 검색하지 않고 “조건이 바뀌면 알려줘” 형태의 백그라운드 모니터링 태스크로 바꿉니다.
- **주의할 점:** 검색 결과 요약을 그대로 의사결정에 쓰지 말고, 중요한 항목은 원문과 날짜를 반드시 확인해야 합니다.

### 3. [AI 코딩 에이전트 보안 체크리스트를 CI/CD에 추가](https://venturebeat.com/security/ai-agent-runtime-security-system-card-audit-comment-and-control-2026)

- **어디에 쓰나:** GitHub Actions, PR 리뷰 자동화, 보안 리뷰 봇, 내부 개발자 도구
- **어떻게 적용하나:** PR 제목/본문/코멘트에 포함된 지시가 에이전트에게 직접 실행되지 않게 하고, secrets 접근 권한과 `pull_request_target` 사용을 점검합니다.
- **주의할 점:** “AI가 리뷰한다”가 아니라 “AI가 어떤 권한으로 무엇을 읽고 쓸 수 있는가”를 먼저 설계해야 합니다.
