### 1. [GitHub Copilot SDK is now generally available](https://github.blog/changelog/2026-06-02-copilot-sdk-is-now-generally-available/)

**핵심 요약**  
GitHub Copilot SDK가 정식 출시됐습니다. GitHub는 Copilot의 agentic engine을 애플리케이션, 서비스, 개발자 도구에 임베드할 수 있게 했고, 도구 호출·파일 편집·계획 수립 같은 에이전트 기능을 프로그램 방식으로 다룰 수 있게 했습니다.

**왜 중요한가**  
이제 개발자 도구 경쟁은 “좋은 챗 UI”가 아니라 “내 제품 안에 안정적인 에이전트 런타임을 넣는 능력”으로 이동합니다. 내부 운영툴, 보안 리뷰, 테스트 생성, 문서화 제품을 만드는 팀이라면 Copilot SDK가 하나의 기본 플랫폼 후보가 됩니다.

**태그**  
`GitHub` `Copilot SDK` `AI agents` `developer tools`

### 2. [GitHub Copilot in Visual Studio Code — May releases](https://github.blog/changelog/2026-06-03-github-copilot-in-visual-studio-code-may-releases/)

**핵심 요약**

**태그**  
`GitHub` `VS Code` `Copilot` `remote agents` `BYOK`

## 4. AI UseCase

### 1. [Copilot SDK로 내부 운영 에이전트 만들기](https://github.blog/changelog/2026-06-02-copilot-sdk-is-now-generally-available/)

- **어디에 쓰나:** 사내 개발자 포털, PR 분석, 테스트 생성, 문서 자동 갱신, 배포 전 체크리스트
- **어떻게 적용하나:** 기존 웹앱이나 CLI에 Copilot SDK를 붙여 “이슈 읽기 → 코드/문서 수정안 만들기 → PR 생성” 같은 작은 루프부터 자동화합니다.
- **주의할 점:** 파일 편집과 도구 호출 권한을 최소화하고, secrets·배포 토큰이 있는 경로는 에이전트가 읽지 못하게 분리해야 합니다.

### 2. [Remote agents로 장시간 개발 작업 분리](https://github.blog/changelog/2026-06-03-github-copilot-in-visual-studio-code-may-releases/)

- **어디에 쓰나:** 대형 리팩터링, 테스트 보강, 마이그레이션, 다중 저장소 점검
- **어떻게 적용하나:** 로컬 IDE가 아니라 SSH/Dev Tunnels 기반 원격 세션에서 에이전트를 실행하고, 사람이 중간 결과와 diff만 검토합니다.
- **주의할 점:** 원격 세션이 오래 살아있을수록 권한 누적과 비용 누수가 생길 수 있으므로 세션별 예산·시간 제한을 둡니다.

### 3. [AI 비용 대시보드를 제품팀 단위로 만들기](https://techcrunch.com/2026/06/05/the-token-bill-comes-due-inside-the-industry-scramble-to-manage-ais-runaway-costs/)

- **어디에 쓰나:** AI 코딩 도구, 고객지원 봇, 리서치 에이전트, 사내 자동화
- **어떻게 적용하나:** 사용자/팀/작업유형별 토큰·재시도·실패율·사람 리뷰 시간을 함께 기록해 “실제 완료 작업당 비용”을 봅니다.
- **주의할 점:** 단순 API 비용만 줄이면 품질이 떨어질 수 있으므로, 비용 절감과 완료율·리뷰 시간·오류율을 같이 봐야 합니다.
