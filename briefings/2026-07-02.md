### 1. [Frontier model 도입 전에 jailbreak severity matrix 만들기](https://www.anthropic.com/news/redeploying-fable-5)

- **어디에 쓰나:** 보안팀 / AI governance / platform engineering / regulated industry
- **어떻게 적용하나:** 모델 도입 전 red-team finding을 severity, 재현 난이도, weaponization 가능성, 영향 범위, fallback 처리 방식으로 분류합니다. 차단된 요청이 어떤 모델로 fallback되는지, 사용자에게 어떤 문구가 보이는지, 감사 로그에 어떤 필드가 남는지도 함께 정의합니다.
- **주의할 점:** vendor의 안전성 설명만으로 내부 risk sign-off를 끝내면 안 됩니다. 조직별 데이터, 권한, threat model이 다르므로 자체 abuse case와 incident playbook을 별도로 둬야 합니다.

### 2. [Copilot browser tools로 UI 회귀 테스트를 에이전트에게 맡기기](https://github.blog/changelog/2026-07-01-browser-tools-for-github-copilot-in-vs-code-are-generally-available/)

- **어디에 쓰나:** 프론트엔드 개발 / QA / design system / customer support tooling
- **어떻게 적용하나:** PR마다 에이전트에게 local dev server를 열고 핵심 페이지를 순회하게 합니다. console error, screenshot, form submission, responsive 상태를 확인하게 하고, 실패한 흐름은 재현 steps와 patch 후보로 남기게 합니다.
- **주의할 점:** 사내 admin, payment, production customer data가 걸린 도메인은 allowlist에서 빼야 합니다. agent tab isolation과 clipboard, location, notification 권한 정책도 팀 표준으로 문서화해야 합니다.

### 3. [업무 agent를 LLM loop가 아니라 graph workflow로 설계하기](https://developers.googleblog.com/announcing-adk-go-20/)

- **어디에 쓰나:** 환불 처리 / 내부 승인 / 고객지원 escalation / 데이터 분석 pipeline
- **어떻게 적용하나:** 항상 같은 순서로 가야 하는 단계는 ADK 같은 graph workflow로 고정하고, 애매한 판단만 LLM node에 맡깁니다. human approval, retry, timeout, branch, fan-out/fan-in, resume point를 workflow에 명시하면 운영 중 실패 원인을 추적하기 쉬워집니다.
- **주의할 점:** 모든 것을 deterministic하게 만들면 예외 처리 폭발이 생깁니다. 반대로 모든 것을 LLM에게 맡기면 비용과 분산이 커집니다. 업무별로 "고정 단계"와 "판단 단계"를 먼저 나누는 설계가 필요합니다.

## 9. 오늘의 관전 포인트

1. **Fable 5 복구 이후 실제 차단·fallback 경험이 어떤가** - safety classifier가 사용자 경험, cyber defense 작업, enterprise logging에 어떤 마찰을 만드는지 봐야 합니다.
2. **Copilot의 model routing이 비용을 정말 낮추는가** - Kimi, auto model selection, session cap이 붙었지만 실제 cost per completed task와 quality regression은 각 팀의 작업군별로 재측정해야 합니다.
3. **agent framework 경쟁이 "prompt loop"에서 "workflow runtime"으로 정리되는가** - Genkit, ADK, Copilot SDK, Claude Code류 도구가 모두 장기 실행, 승인, subagent, browser use, 비용 cap을 제품 표면에 올리고 있습니다.
4. **AI compute 공급자가 cloud vendor로 바뀌는가** - SpaceX와 Meta가 compute 판매를 검토·확대하는 흐름은 모델 회사와 hyperscaler 사이의 시장 경계를 흐립니다.
