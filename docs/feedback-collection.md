# Feedback Policy

v0에서는 좋아요 수집, 폼, 버튼, 자동 추천 학습을 **하지 않는다**.

## 이유

사용자 피드백:

> 좋아요 모으는 거 빼자. 이거 너무 복잡해진다.

이 결정에 따라 v0는 다시 “끝나는 실험”에 집중한다.

## v0 운영 규칙

- 브리핑 본문에 좋아요 버튼을 넣지 않는다.
- Tally / Google Form / Airtable / Cloudflare Worker 같은 피드백 수집 백엔드를 붙이지 않는다.
- `{{LIKE_URL}}` placeholder를 사용하지 않는다.
- 피드백은 사용자가 대화에서 자연스럽게 주는 내용만 기록한다.
- 자동 개인화 추천은 v0 범위 밖이다.

## 유지하는 것

- 중복 방지: `data/published_items.csv`
- 날짜별 브리핑 파일: `briefings/YYYY-MM-DD.md`
- 사용자 피드백 메모: `data/feedback.csv`
- 제목 링크, GeekNews 개별 링크, 관련 출처 최소화 원칙

## 나중에 다시 검토할 수 있는 조건

좋아요 수집은 아래 조건이 충족될 때만 다시 검토한다.

1. 7일 브리핑 실험이 끝남
2. 브리핑을 계속 운영하기로 결정함
3. 블로그 게시 플랫폼이 정해짐
4. 수동 피드백만으로는 선호 추적이 부족하다고 판단됨

그 전까지는 좋아요/추천/자동 수집을 만들지 않는다.
